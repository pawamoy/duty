53 """Run `flake8`.

55 Parameters: 

56 *paths: Paths to check. 

57 config: Path to the config file that will be the authoritative config source. 

58 This will cause Flake8 to ignore all other configuration files. 

59 verbose: Print more information about what is happening in flake8. 

60 This option is repeatable and will increase verbosity each time it is repeated. 

61 output_file: Redirect report to a file. 

62 append_config: Provide extra config files to parse in addition to the files found by Flake8 by default. 

63 These files are the last ones read and so they take the highest precedence when multiple files provide the same option. 

64 isolated: Ignore all configuration files. 

65 enable_extensions: Enable plugins and extensions that are otherwise disabled by default. 

66 require_plugins: Require specific plugins to be installed before running. 

67 quiet: Report only file names, or nothing. This option is repeatable. 

68 color: Whether to use color in output. Defaults to `auto`. 

69 count: Print total number of errors to standard output and set the exit code to 1 if total is not empty. 

70 exclude: Comma-separated list of files or directories to exclude (default: ['.svn', 'CVS', '.bzr', '.hg', '.git', '__pycache__', '.tox', '.nox', '.eggs', '*.egg']). 

71 extend_exclude: Comma-separated list of files or directories to add to the list of excluded ones. 

72 filename: Only check for filenames matching the patterns in this comma-separated list (default: ['*.py']). 

73 stdin_display_name: The name used when reporting errors from code passed via stdin. This is useful for editors piping the file contents to flake8 (default: stdin). 

74 error_format: Format errors according to the chosen formatter. 

75 hang_closing: Hang closing bracket instead of matching indentation of opening bracket's line. 

76 ignore: Comma-separated list of error codes to ignore (or skip). For example, ``--ignore=E4,E51,W234`` (default: E121,E123,E126,E226,E24,E704,W503,W504). 

77 extend_ignore: Comma-separated list of error codes to add to the list of ignored ones. For example, ``--extend-ignore=E4,E51,W234``. 

78 per_file_ignores: A pairing of filenames and violation codes that defines which violations to ignore in a particular file. The filenames can be specified in a manner similar to the ``--exclude`` option and the violations work similarly to the ``--ignore`` and ``--select`` options. 

79 max_line_length: Maximum allowed line length for the entirety of this run (default: 79). 

80 max_doc_length: Maximum allowed doc line length for the entirety of this run (default: None). 

81 indent_size: Number of spaces used for indentation (default: 4). 

82 select: Comma-separated list of error codes to enable. For example, ``--select=E4,E51,W234`` (default: E,F,W,C90). 

83 extend_select: Comma-separated list of error codes to add to the list of selected ones. For example, ``--extend-select=E4,E51,W234``. 

84 disable_noqa: Disable the effect of "# noqa". This will report errors on lines with "# noqa" at the end. 

85 show_source: Show the source generate each error or warning. 

86 no_show_source: Negate --show-source. 

87 statistics: Count errors. 

88 exit_zero: Exit with status code "0" even if there are errors. 

89 jobs: Number of subprocesses to use to run checks in parallel. This is ignored on Windows. The default, "auto", will auto-detect the number of processors available to use (default: auto). 

90 tee: Write to stdout and output-file. 

91 benchmark: Print benchmark information about this run of Flake8. 

92 bug_report: Print information necessary when preparing a bug report. 

93 

94 Returns: 

95 Success/failure.
