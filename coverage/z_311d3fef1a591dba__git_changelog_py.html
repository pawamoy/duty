41 r"""Run `git-changelog`. 

42 

43 Parameters: 

44 repository: The repository path, relative or absolute. Default: current working directory. 

45 config_file: Configuration file(s). 

46 bump: Specify the bump from latest version for the set of unreleased commits. 

47 Can be one of `auto`, `major`, `minor`, `patch` or a valid SemVer version (eg. 1.2.3). 

48 For both SemVer and PEP 440 versioning schemes (see -n), `auto` will bump the major number 

49 if a commit contains breaking changes (or the minor number for 0.x versions, see -Z), 

50 else the minor number if there are new features, else the patch number. Default: unset (false). 

51 versioning: Versioning scheme to use when bumping and comparing versions. 

52 The selected scheme will impact the values accepted by the `--bump` option. 

53 Supported: `pep440`, `semver`. 

54 

55 PEP 440 provides the following bump strategies: `auto`, `epoch`, `release`, `major`, `minor`, `micro`, `patch`, 

56 `pre`, `alpha`, `beta`, `candidate`, `post`, `dev`. 

57 Values `auto`, `major`, `minor`, `micro` can be suffixed with one of `+alpha`, `+beta`, `+candidate`, and/or `+dev`. 

58 Values `alpha`, `beta` and `candidate` can be suffixed with `+dev`. 

59 Examples: `auto+alpha`, `major+beta+dev`, `micro+dev`, `candidate+dev`, etc.. 

60 

61 SemVer provides the following bump strategies: `auto`, `major`, `minor`, `patch`, `release`. 

62 See the docs for more information. Default: unset (`semver`). 

63 in_place: Insert new entries (versions missing from changelog) in-place. 

64 An output file must be specified. With custom templates, you can pass two additional 

65 arguments: `--version-regex` and `--marker-line`. 

66 When writing in-place, an `in_place` variable will be injected in the Jinja context, 

67 allowing to adapt the generated contents (for example to skip changelog headers or footers). 

68 Default: unset (false). 

69 version_regex: A regular expression to match versions in the existing changelog 

70 (used to find the latest release) when writing in-place. 

71 The regular expression must be a Python regex with a `version` named group. 

72 Default: `^## \[(?P<version>v?[^\]]+)`. 

73 marker_line: A marker line at which to insert new entries (versions missing from changelog). 

74 If two marker lines are present in the changelog, the contents between those two lines 

75 will be overwritten (useful to update an 'Unreleased' entry for example). Default: `<!-- insertion marker -->`. 

76 output: Output to given file. Default: standard output. 

77 provider: Explicitly specify the repository provider. Default: unset. 

78 parse_refs: Parse provider-specific references in commit messages (GitHub/GitLab/Bitbucket issues, PRs, etc.). 

79 Default: unset (false). 

80 release_notes: Output release notes to stdout based on the last entry in the changelog. Default: unset (false). 

81 input: Read from given file when creating release notes. Default: `CHANGELOG.md`. 

82 convention: The commit convention to match against. Default: `basic`. 

83 sections: A comma-separated list of sections to render. 

84 See the available sections for each supported convention in the description. Default: unset (None). 

85 template: The Jinja2 template to use. 

86 Prefix it with `path:` to specify the path to a Jinja templated file. Default: `keepachangelog`. 

87 git_trailers: Parse Git trailers in the commit message. 

88 See https://git-scm.com/docs/git-interpret-trailers. Default: unset (false). 

89 omit_empty_versions: Omit empty versions from the output. Default: unset (false). 

90 no_zerover: By default, breaking changes on a 0.x don't bump the major version, maintaining it at 0. 

91 With this option, a breaking change will bump a 0.x version to 1.0. 

92 filter_commits: The Git revision-range filter to use (e.g. `v1.2.0..`). Default: no filter. 

93 jinja_context: Pass additional key/value pairs to the template. 

94 Option can be used multiple times. 

95 The key/value pairs are accessible as 'jinja_context' in the template. 

96 version: Show the current version of the program and exit. 

97 debug_info: Print debug information.
