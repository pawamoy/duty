42 r"""Run `black`. 

43 

44 Parameters: 

45 src: Format the directories and file paths. 

46 config: Read configuration from this file path. 

47 code: Format the code passed in as a string. 

48 line_length: How many characters per line to allow [default: 120]. 

49 target_version: Python versions that should be supported by Black's output. 

50 By default, Black will try to infer this from the project metadata in pyproject.toml. 

51 If this does not yield conclusive results, Black will use per-file auto-detection. 

52 check: Don't write the files back, just return the status. Return code 0 means nothing would change. 

53 Return code 1 means some files would be reformatted. Return code 123 means there was an internal error. 

54 diff: Don't write the files back, just output a diff for each file on stdout. 

55 color: Show colored diff. Only applies when `--diff` is given. 

56 fast: If --fast given, skip temporary sanity checks. [default: --safe] 

57 pyi: Format all input files like typing stubs regardless of file extension 

58 (useful when piping source on standard input). 

59 ipynb: Format all input files like Jupyter Notebooks regardless of file extension 

60 (useful when piping source on standard input). 

61 python_cell_magics: When processing Jupyter Notebooks, add the given magic to the list of known python-magics 

62 (capture, prun, pypy, python, python3, time, timeit). Useful for formatting cells with custom python magics. 

63 skip_source_first_line: Skip the first line of the source code. 

64 skip_string_normalization: Don't normalize string quotes or prefixes. 

65 skip_magic_trailing_comma: Don't use trailing commas as a reason to split lines. 

66 preview: Enable potentially disruptive style changes that may be added 

67 to Black's main functionality in the next major release. 

68 quiet: Don't emit non-error messages to stderr. Errors are still emitted; silence those with 2>/dev/null. 

69 verbose: Also emit messages to stderr about files that were not changed or were ignored due to exclusion patterns. 

70 required_version: Require a specific version of Black to be running (useful for unifying results 

71 across many environments e.g. with a pyproject.toml file). 

72 It can be either a major version number or an exact version. 

73 include: A regular expression that matches files and directories that should be included on recursive searches. 

74 An empty value means all files are included regardless of the name. Use forward slashes for directories 

75 on all platforms (Windows, too). Exclusions are calculated first, inclusions later [default: (\.pyi?|\.ipynb)$]. 

76 exclude: A regular expression that matches files and directories that should be excluded on recursive searches. 

77 An empty value means no paths are excluded. Use forward slashes for directories on all platforms (Windows, too). 

78 Exclusions are calculated first, inclusions later [default: /(\.direnv|\.eggs|\.git|\.hg|\.mypy_cache|\.nox| 

79 \.tox|\.venv|venv|\.svn|\.ipynb_checkpoints|_build|buck-out|build|dist|__pypackages__)/]. 

80 extend_exclude: Like --exclude, but adds additional files and directories on top of the excluded ones 

81 (useful if you simply want to add to the default). 

82 force_exclude: Like --exclude, but files and directories matching this regex will be excluded 

83 even when they are passed explicitly as arguments. 

84 stdin_filename: The name of the file when passing it through stdin. Useful to make sure Black will respect 

85 --force-exclude option on some editors that rely on using stdin. 

86 workers: Number of parallel workers [default: number CPUs in the system].
